Для начала прочитайте [[Создание структуры папки resources|гайд]] по созданию структуры папки **resources**.
Теперь зайдите в папку resources.data.recipes и создайте там json файл с произвольным названием (но желательно похожим на название крафтового предмета, для избежания путаницы). Теперь зайдите в этот файл и создайте там примерно вот такую структуру:
```json
{  
  "type": "minecraft:crafting_shaped",  
  "category": "misc",  
  "show_notification": true,
  "key": {  
    "P": {  
      "item": "minecraft:diamond"  
    }  
  },  
  "pattern": [  
    "P  ",  
    "P  ",  
    "PPP"  
  ],  
  "result": {  
    "item": "minecraft:nether_star",  
    "count": 1  
  }
}
```

## Теперь давайте пройдёмся по полям:
### "type"
```json
{"type": "minecraft:crafting_shaped"}
```
здесь мы указываем тип крафта - для верстака существует всего 2 таких типа:
1) crafting_shaped
2) crafting_shapless
подробнее второй я разберу чуть ниже.

### "category"
```json
{"category": "misc"}
```
в этом поле мы указываем вкладку, где этот предмет будет находится в книге рецептов.
в книге рецептов также существуют такие вкладки: 
- building
- equipment 
- misc
- redstone
- combat
- tools
- transportation

### "show_notification"
Поле "show_notification" контролирует появление всплывающего уведомления в чате при разблокировке нового рецепта - если включено, то при получении всех ингредиентов в первый раз вылезет подсказка о новом доступном крафте.

### "key"
В этом поле мы указываем список из ключей, в которых присваиваем значения предмета для крафта определённым символам (ключ состоит из 1 символа!), для последующего использования в шаблоне. 

**!!!ВАЖНО** все ключи должны использоваться в шаблоне - если ключ существует, но не появляется в шаблоне, то рецепт **НЕ** появится в игре.

если нам нужно несколько ключей (несколько разных предметов в крафте), то нужно указать их через запятую в таком формате:
```json
"key": {  
    "P": {  
      "item": "minecraft:diamond"  
    },
    "K": {
	  "item": "minecraft:stone"
    }  
  }  
```

### "pattern"
В этом поле мы уже непосредственно расставляем предметы, которые мы присвоили в ключи, в сетку крафта.
думаю что на примере и так всё понятно - мы ставим символы в той форме, в какой хотели бы видеть их на верстаке (пробел для пустого слота).

Но есть пара важны моментов:
#### 1: внимательно с пробелами
если у вас в крафте **максимальная** ширина рецепта 3, то в каждой строчке должна быть такая-же длина. такую запись, например, майнкрафт не примет (рецепт просто не будет работать):
```json
"pattern": [  
    "P",  
    "P",  
    "PPP"  
  ]
```

просто делайте так, чтобы поле патерна всегда было квадратным - и проблем не будет.
#### 2: поле **можно** масштабировать
например если у вас крафт из 6 предметов 2 на 3, то вы **можете** записать это так:
```json
"pattern": [  
    "PP",  
    "PP",  
    "PP"  
  ]
```
или наоборот так:
```json
"pattern": [  
    "PPP",  
    "PPP"  
  ]
```
может показаться, что это на что-то влияет - например на строгость расположения рецепта в сетке верстака, однако тут мы подходим к самому главному **НО**:
#### 3: такой рецепт **ВСЕГДА** подвижный
если бы вы сделали подобный рецепт:
```json
"pattern": [  
    "PP",  
    "PP"  
  ]
```
то он бы работал в любом из углов верстака - он не будет чётко привязан к какому-то углу.
даже если вы сделаете так:
```json
"pattern": [  
    "PP ",  
    "PP ",
    "   "  
  ]
```
это ничего не изменит. это создаёт некоторые очень уникальные проблемы, которые я видел даже в одном очень популярном моде.
при этом внутри json файла это **никак** нельзя исправить - способ есть, но через уже **java** код - о нём позже.

### "result" 
это список, определяющий что и сколько мы получим в результате крафта.
в неё так-же есть 2 поля:
#### "item"
здесь нужно указать код предмета который мы хотим получать при крафте
#### "count" 
здесь нужно указать целое число - количество предметов которые игрок будет получать при крафте. здесь **нельзя** указать рандомное значение через "uniform", однако можно указать значение больше 64 (с чем интересно поэкспериментировать)
## crafting_shapless
этот тип отличается тем, что в нём предметы могут находится в верстаке в **любом** порядке, что в некоторых случаях удобнее чем обычный шаблон.
для начала я покажу пример работы такого типа крафта прямо из майнкрафта:
```json
{
  "type": "minecraft:crafting_shapeless",
  "category": "misc",
  "ingredients": [
    { "item": "minecraft:paper" },
    { "item": "minecraft:paper" },
    { "item": "minecraft:paper" },
    { "item": "minecraft:leather" }
  ],
  "result": {
    "item": "minecraft:book",
    "count": 1
  }
}
```
как вы можете заметить, поля "key" и "pattern" заменились на "ingredients".
в этом поле нам нужно указать массив предметов, которые должны находится на верстаке для крафта. при этом, если нужно несколько одинаковых предметов, то просто укажите их несколько раз (как в примере)
