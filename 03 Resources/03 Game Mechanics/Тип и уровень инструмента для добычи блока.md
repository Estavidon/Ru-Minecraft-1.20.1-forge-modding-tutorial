Если у вас ещё нет готовой структуры папки resources, то создайте её по этому [[Создание структуры папки resources|гайду]].
Для начала зададим тип инструмента, которым можно будет добывать ваш блок. Для этого перейдите в папку **resources.data.minecraft.tags.blocks.mineable** и создайте там файл **pickaxe.json**. Затем перейдите в него и создайте простую структуру:
```json
{
	"values": [
		"ваш mod:код блока",
		"myfirstmod:block1" // пример
	]
}
```
Теперь все блоки, указанные здесь, будут добываться этим инструментом. А если при создании блока вы использовали в билдере .requiresCorrectToolForDrops() или скопировали свойства блока с этим методом, то указанный блок **не** будет добываться другими (не помеченными) типами инструментов. 
Сейчас мы создавали таблицу добычи для **кирки** — файл так и называется в переводе с английского. А чтобы создать таблицу для других инструментов, нужно соответственно создать другой файл в формате: **название инструмента.json**. А вот список инструментов, доступных в ванильном Minecraft:
1) axe
2) hoe
3) pickaxe
4) shovel
5) shears
6) sword

А чтобы добавить уровень добычи для выбранного инструмента, зайдите в папку **resources.data.minecraft.tags.blocks** и создайте файл **needs_iron_tool.json**. Затем перейдите в него и создайте аналогичную структуру:
```json
{
	"values": [
		"ваш mod:код блока",
		"myfirstmod:block1" // пример
	]
}
```
Теперь вы можете добавить любой блок сюда — это задаёт его уровень инструмента. В данном случае это железный (например, блок падает при добыче киркой не ниже железной).
Следовательно, название файла образуется так: **needs_уровень добычи_tool.json**.
В ванильном Minecraft существуют следующие уровни добычи:
1) stone
2) iron
3) diamond
Однако тут нет деревянного — это объясняется тем, что если **не** добавлять предмет ни в один из уровней добычи, то он будет добываться **любым** инструментом нужного типа (то есть и деревянным тоже).
